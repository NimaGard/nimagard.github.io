In the Pursuit of Temporal Features

Representation Learning World Models Robot Learning

A robot arm reaches for a mug. It has seen this mug before in a thousand training videos, being grasped, lifted, placed. But today the mug is wet. The hand slips. The mug begins to fall. The robot has 200 milliseconds to respond, which is not enough time to re-plan from scratch, but enough to catch the mug if it understands that falling is happening. Not what the mug looks like (spatial features), but what it is doing (temporal features).

This distinction between what and what happens sits at the heart of every challenge in video understanding, temporal consistency in generation, action prediction for control, physics simulation for planning, and more recently world modeling and memory. They all require a model to have some internal representation of how the world changes across time. Do today's video models actually represent time, or do they just process frames?

Consider these two images. They are identical. One is the other rotated 180 degrees. In the left image, the ball seems to be tossed upward. In the right, it appears to fall. A single frame is fundamentally ambiguous about dynamics. A video would resolve this instantly, but only if the model can read the temporal structure of the frames, not just their spatial content.

What are temporal features?

Think of a filmstrip where each frame captures spatial features such as objects, textures, colors, and geometry. The whole sequence, though, tells a story of how the world changes across time. Spatial features are everything you could learn from a single frame: there is a hand, there is a ball, the background is blue. Temporal features are what emerges from the sequence: the ball is moving upward, the hand is opening, the ball will leave the hand in two frames. These aren't properties of any single frame; they're properties of transitions between frames.

When we think about temporal features, there are two questions that come to mind:

  • Are temporal features learned dynamics? In other words, are temporal features technically a function that evolves the spatial features due to an action? \(z_{t+1} = f(z_t | a_t)\), where \(f(.)\) is the temporal feature.
  • Can temporal features exist without spatial features? Are temporal features just like any other type of feature, but dependent on the spatial features? For instance, is picking up a cup different from picking up an orange?

Consider a classical example: spatial features can be extracted from individual frames using techniques like the Sobel operator, while temporal features can be captured using optical flow. Optical flow is a hand-crafted temporal feature that takes the spatial features from two consecutive frames and outputs how those features move from one frame to the next.

Now let's look at what modern video models actually learn. I fed the same robot manipulation video into four pretrained models and visualized their internal features using PCA. Each model's high-dimensional hidden states are projected down to 3 components and mapped to RGB. Regions with the same color have similar features; color changes across frames reveal temporal dynamics.

It's clear that spatial features from modern and classical models are quite different. Some are more abstract, and some are more object-like. This raises a natural question: can we organize temporal features into a similar hierarchy?

An abstraction ladder for temporal features

Spatial features have an established abstraction hierarchy: edges → textures → parts → objects → scenes, where each level builds on the one below. Temporal features don't have an equivalent canonical ladder yet, but we can sketch one by looking at how the field has evolved.

Level What it encodes Example Spatial parallel
1. Motion Pixel displacement between frames Optical flow Edges
2. Spatiotemporal patterns Local motion+appearance primitives 3D conv filters (●) Textures
3. Cross-frame relations Correspondence across time Video transformer attention (● ●) Parts
4. Predictive features Representations enabling future prediction Masked video modeling (●) Objects
5. Dynamics Latent physics / world model Inverse dynamics module (●) Scene understanding

At Level 1, temporal features are just motion: how pixels shift between frames. Classical optical flow lives here. At Level 2, 3D convolutions learn local spatiotemporal primitives like a moving edge or a closing gripper; Wan 2.1's causal-conv VAE operates at this level. Level 3 is where video transformers sit. TimeSformer and ViViT learn cross-frame attention patterns, but their extremely low-rank PCA (90–96% variance in 3 components) suggests they compress away most temporal structure in favor of classification-relevant features.

Level 4 is where self-supervised models like VJEPA2 operate. They learn representations that make the future predictable, and VJEPA2's PCA captures only 22% of the variance in 3 components because the information is distributed across a high-dimensional manifold encoding dynamics, not just appearance.

Level 5—latent physics—is where temporal features are not trajectories but the laws governing transitions: \(z_{t+1} = f(z_t, a_t)\), where \(f\) encodes how actions transform states. Perhaps this is the level we need in robotics: not "the cup moved right" but "pushing a rigid object on a smooth surface produces predictable sliding."

Spatial features represent what exists.
Temporal features represent how the world changes.

The question, then, is not whether videos contain temporal information (they obviously do), but whether neural networks that process videos build separate internal representations for temporal and spatial features, or whether these are hopelessly entangled—and at which level of this ladder each model actually operates.

How models process video

There are broadly four architectural approaches to feeding video into a neural network. Each makes a different implicit assumption about the relationship between time and space.

01

Frame-by-frame + temporal aggregation

Treat each frame independently through a 2D backbone, reshape to (B·T) C H W, then aggregate temporally with an RNN, LSTM, or temporal pooling.

ConvLSTM, early action recognition
02

3D convolution

Treat the video as a 3D volume B C T H W and apply 3D convolutions that span space and time simultaneously. A step in time is equivalent to a step in space.

Wan 2.1 VAE, Video Swin
03

Factored attention

Alternate between spatial attention (within a frame) and temporal attention (across frames). Architecturally separates the two axes.

TimeSformer, Genie ST-ViViT, ViViT
04

Unified spatiotemporal attention

All tokens from all frames attend to all other tokens. No architectural distinction between time and space. Positional encoding is the only signal.

VJEPA2

The natural assumption is that approach 3 (factored attention) should produce the cleanest separation of temporal and spatial features, since it literally processes them in separate attention layers. Approach 4 seems like it would mix everything together. But is that actually what happens?

The experiment

I probed four pretrained video models from three different families, with different architectures and training objectives, to answer this question: are temporal features disentangled from spatial features in these representations?

Model Architecture Position encoding Approach
● Wan 2.1 3D causal conv (VAE) + DiT 3D RoPE 02 + 04
● VJEPA2 ViT-Large, unified attention 3D RoPE 04
● TimeSformer Divided space-time attention Learned (separate) 03
● ViViT ViT-Base, unified attention Learned (unified) 04

Three probes, each testing a different aspect of temporal disentanglement:

  1. Position encoding ablation. Zero out the temporal position encoding and measure how much the output changes. If temporal positions carry information, this should cause a large shift.
  2. Frame shuffle sensitivity. Randomly reorder the video frames and compare hidden states at each layer. If a model encodes temporal order, shuffling should destroy that signal—and we can track where in the network this matters most.
  3. Attention head specialization. For models with accessible attention weights, measure whether individual heads preferentially attend across time vs. across space.

Results

I removed temporal position encodings from each model and measured the cosine similarity between the original and ablated outputs. A value of 1.0 means the ablation had no effect; lower values mean the model relied on that positional information.

Removing temporal position encodings from TimeSformer and ViViT does nothing: cosine similarity stays at 0.999 and 1.000, meaning these models no longer depend on their temporal positional signals at inference. VJEPA2, by contrast, drops to 0.630. Its 3D RoPE temporal dimensions carry substantial information that the model actively uses. Wan 2.1's DiT shows a milder effect at 0.928 on synthetic patterns, where its RoPE matters less when temporal structure is minimal.

Perhaps what determines temporal feature persistence is not the attention architecture (factored vs. unified) but the position encoding mechanism. Additive learned embeddings are absorbed during training. The model learns to rely on content instead, even though they measurably improve training. Multiplicative RoPE is re-injected at every layer, making temporal position an inextricable part of the attention computation.

Why the difference? Learned position embeddings are additive: summed into token representations at the input and never re-injected. After dozens of layers of LayerNorm and residual connections, the network can learn to compensate, and the positional signal fades into the content representation. RoPE works differently. It applies rotation matrices to the query and key vectors at every attention layer, so the dot product \(q_i \cdot k_j\) is multiplicatively modulated by the relative position \((i - j)\). Temporal position isn't just a hint added to the input; it's structural infrastructure that shapes every attention score the model computes. The network cannot learn around it because it's re-injected at every layer.

There is a deeper reason RoPE produces rich temporal features. Each query-key dimension rotates at a different frequency, a direct consequence of the sinusoidal basis. Lower dimensions rotate fast (high frequency, short period) and complete full cycles within the sequence; upper dimensions rotate slowly (low frequency, long period) and may only traverse a monotonic arc. This creates a natural division of labor: high-frequency dimensions act like a fine-grained temporal hash, distinguishing nearby frames but aliasing distant ones, while low-frequency dimensions preserve monotonic ordering across the full clip, capturing long-range temporal dependencies. Ablating the temporal RoPE axes therefore doesn't remove a single signal—it simultaneously destroys both the local frame-to-frame signatures and the global temporal ordering that the model has learned to rely on.

To see this concretely, consider VJEPA2's 3D RoPE. Each head's 64 dimensions are split into three groups: 20 for temporal position, 20 for height, and 20 for width. Here is what those position encodings look like across all 2,048 tokens (8 temporal × 16 × 16 spatial):

Zeroing out the temporal RoPE dimensions not only removes a signal but also corrupts the rotation matrices, scrambling the attention scores between tokens at different timesteps. This is why the effect is so large (cosine similarity drops to 0.630): the model's learned attention patterns are predicated on those rotations being present. Spatial ablation, by contrast, causes significant disruption in all models, regardless of encoding type. Spatial layout carries non-redundant structural information (edges, objects, part relationships) that content features alone cannot reconstruct, so even additive spatial embeddings resist absorption. Spatial positions, then, are universally important, but can we say the same for temporal positions?

Validation on real robot videos

Synthetic stimuli (moving bars, brightness ramps) are useful for controlled experiments, but real videos could behave differently. I reran the same probes on real robot manipulation videos.

The pattern holds on real videos. In some cases it's even more extreme. VJEPA2's temporal RoPE ablation drops to 0.558 (vs. 0.630 on synthetic), while TimeSformer and ViViT remain unmoved. Wan 2.1's DiT sits in between at 0.838. Its temporal RoPE matters, but less than VJEPA2's, likely because the DiT also receives strong content signal from the noisy latents and prompt conditioning. Those latents already carry temporal structure: the upstream 3D causal-conv VAE is order-sensitive, with cosine distance ~0.34 between original and shuffled reconstructions, so the DiT builds on a temporally-aware foundation. The conclusion still holds: this is not an artifact of synthetic stimuli.

VJEPA2 shows an interesting pattern: similarity starts high in early layers (the model hasn't yet built temporal-order-dependent features) and then steadily drops through deeper layers, falling to 0.63 at the final layer. The network progressively builds representations that depend on frame order. TimeSformer and ViViT show the opposite trend: similarity is lower in early layers and increases toward the output, meaning these models are gradually removing temporal order information through depth, not accumulating it. Their final representations are largely order-invariant. Wan 2.1 VAE shows flat sensitivity around 0.65-0.76, dipping to 0.63 at the output, indicating mild sensitivity to frame order on simple stimuli.

The bottom row is the punchline. Despite different architectures—factored attention, unified attention, 3D convolutions, RoPE, learned embeddings—none of these models achieves functional temporal disentanglement. In every case, temporal and spatial information are entangled in the final representation.

What this means

1. Factored attention helps accuracy, not disentanglement. TimeSformer's divided attention achieves better classification accuracy than joint attention, especially on temporal-heavy benchmarks like SSv2. But this does not mean its temporal and spatial features are separable in the representation. Factored processing produces better features, not disentangled ones.

2. The position encoding mechanism determines inference-time dependence. RoPE injects positional information multiplicatively into attention scores, making it structurally impossible for the model to absorb away. Learned additive position embeddings shape training (they improve accuracy), but their signal gets internalized into content features, leaving no inference-time dependence. If you need a model to explicitly and persistently represent temporal position, RoPE seems to be the way to go.

3. Content-based temporal understanding may be sufficient at inference. TimeSformer and ViViT achieve competitive performance on action recognition benchmarks while showing zero inference-time dependence on temporal position encodings. Temporal position embeddings helped them learn content-based temporal reasoning during training, but at inference the model reads temporal order purely from what things look like across frames. This is arguably more robust: the temporal understanding has been baked into the weights rather than relying on a positional signal that could be corrupted. Another explanation for this is what we see in registers for vision transformers. Maybe these embeddings are just a garbage collector that you can throw away at inference!

What's next?

There is still an open question in my mind that I didn't investigate in my experiments. We know that the training objective is critical in representation learning. Tasks such as classification and segmentation are all supervised, yet JEPA, the model with the best feature specialization, was trained in a self-supervised way. Maybe the self-supervised training objective is more important for temporal feature representation than the supervised one. Looking into how training objectives shape temporal features is great future work.

And although in our experiments RoPE helped feature specialization, it's not clear that the spatial-temporal decomposition is the right one. Maybe there is a more abstract way to think about features themselves!
